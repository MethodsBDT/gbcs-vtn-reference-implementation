# OpenADR 3 Virtual Top Note (VTN) Reference Implementation

## Requirements
Python 3.5.2+ (tested on 3.9.0)

## Running locally with Python
To run the server, execute the following from the root directory:

```
virtualenv venv
source venv/bin/activate
pip3 install -r requirements.txt
python -m swagger_server
```

## Building and Running with Docker

To run the server in a Docker container, execute the following from the root directory:

```bash
# building the image
docker build -t swagger_server .

# starting up a container
docker run -p 8080:8080 swagger_server
```

## Interacting with the running VTN

The VTN listens for requests on

	http://localhost:8080/openadr3/3.0.1

Currently, the VTN only uses HTTP, not HTTPS, and
it is not necessary to obtain an API token via `POST /auth/token`,
two pre-configured tokens are available for test requests:

* `bl_token`
* `ven_token`

The VTN will need to be "loaded" with OpenADR 3 objects (programs, events, etc) via
the Business Logic (BL) client endpoints (using the `bl_token` above),
subsequently VEN client endpoints/requests may be invoked to retrieve these objecs.

The [openadr3-test-tool](https://github.com/oadr3-org/openadr3-test-tool) repo contains a number of useful test scripts.

## Implementation Details

This server was generated by the [swagger-codegen](https://github.com/swagger-api/swagger-codegen) project. By using the OpenADR 3
[OpenAPI-Spec](https://github.com/swagger-api/swagger-core/wiki) from a remote HTTP server, one can easily generate a server stub.
This server uses the resulting server stub with Flask and the [Connexion](https://github.com/zalando/connexion) library on top of Flask.
